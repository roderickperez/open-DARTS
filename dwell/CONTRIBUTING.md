# Contributing to this repository

Thank you for considering contributing to this project! Your help is greatly appreciated. Please follow the guidelines below to ensure a smooth collaboration.

## How to contribute

### 1. Reporting issues
- Before submitting a new issue, check if it already exists in the [issue tracker](https://gitlab.com/open-darts/dwell/-/issues).
- Provide a clear and concise title.
- Describe the problem with as much detail as possible, including steps to reproduce it.
- If applicable, include screenshots or logs to help diagnose the issue.

### 2. Submitting code changes
#### Fork and clone the repository
- Fork the repository to your GitLab account.
- Clone your fork locally:
  ```sh
  git clone https://gitlab.com/your-username/your-repo.git
  cd your-repo
  ```
- Create a new branch for your changes:
  ```sh
  git checkout -b feature-branch-name
  ```

#### Code guidelines
- Follow the coding style used in the project.
- Ensure that your code is well-documented and formatted.
- Write unit tests if applicable.

#### Committing changes
- Use meaningful commit messages:
  ```sh
  git commit -m "Fix well segment indexing in pipe_model.py"
  ```
- Push your branch to your fork:
  ```sh
  git push origin feature-branch-name
  ```

#### Creating a merge request (MR)
- Go to your repository on GitLab and create a **Merge Request (MR)**.
- Provide a clear title and description.
- Link any related issues.
- Request a review from the maintainers.

### 3. Code review process
- Be responsive to feedback on your MR.
- Make necessary changes and update your branch.
- Once approved, your changes will be merged into the main branch.
